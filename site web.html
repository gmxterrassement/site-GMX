Déployer ton site (zéro expérience requise)

Option A — Vercel (recommandé, super simple)

Vercel s’occupe de tout (build + hébergement). Tu n’as rien à installer sur ton ordi.

  1. Crée un compte sur Vercel (gratuit) : vercel.com

  2. Crée un dépôt GitHub (github.com → New → Repository), puis dépose le contenu du .zip que tu viens de télécharger :

    • Décompresse gmx-site.zip

    • Sur GitHub : Add file → Upload files → glisse tous les fichiers du dossier gmx-site

  3. Sur Vercel : Add New → Project → Import Git Repository → sélectionne ton repo GitHub.

  4. Laisse les valeurs automatiques (Vercel détecte Vite) :

    • Build command : npm run build

    • Output (Publish) directory : dist

  5. Clique Deploy → après quelques secondes, tu obtiens une URL en ligne (ex. https://gmx-site.vercel.app).

➡️ Pour mettre à jour le site : modifie des fichiers, refais Commit sur GitHub → Vercel redéploie automatiquement.

Option B — Netlify

Netlify est aussi très simple.

Avec Git (recommandé)

  1. Compte Netlify : app.netlify.com → Add new site → Import from Git

  2. Choisis ton dépôt GitHub.

  3. Build command: npm run build | Publish directory: dist

  4. Deploy → tu obtiens une URL en ligne.

Sans Git (drag & drop)

  • Il faut d’abord construire le site pour générer un dossier dist.

  • Si tu n’as jamais installé Node, suis “Tester en local” ci-dessous, puis glisse le contenu du dossier dist/ dans Netlify Drop (app.netlify.com/drop).

Option C — GitHub Pages (statique)

  1. Sur ton ordi, construis le site (voir section “Tester en local”) → ça crée un dossier dist.

  2. Publie dist sur la branche gh-pages (ou utilise un plugin de déploiement).

  3. Active Pages dans les Settings → Pages de ton repo, source = gh-pages.

Tester en local (si tu veux voir avant de publier)

  2. Ouvre un Terminal dans le dossier gmx-site puis lance :
